Definición general

definicion

El Portocolo HTTPes el protocolo que permite la comunicación entre un cliente (navegador) y un servidor web.

Su función principal es transferir información como páginas, imágenes o videos a través de Internet

Trabaja bajo el modelo cliente-servidor: el cliente hace una petición y el servidor envía una respuesta.

Estructura y funcionamiento

Estructura y funcionamiento

Definicion:Una comunicación HTTP se compone de dos partes.

Petición (Request) → enviada por el cliente. Contiene:

    Parte1

  • Metodo (Get,post,etc)
  • Url del recurso
  • Encabezados ( informacion adicional)
  • Cuerpo(opcional,cuando se envia datos)

Parte2

  • Codigo de estado (ej.200 ok,404 Not Fonund)
  • Encabezasos del servidor
  • Cuerpo con el contenido solicitado

    • Estructura de una petición HTTP

    Línea de petición: indica el método (GET), el recurso (/index.html) y la versión del protocolo (HTTP/1.1)

    Cabeceras (headers): proporcionan información adicional (navegador, tipo de contenido, idioma, etc.).

    Cuerpo (body): opcional; se usa para enviar datos (por ejemplo, en formularios o JSON)

    ejemplo

    Línea de estado: contiene la versión de HTTP, el código de estado y su descripción (por ejemplo, 200 OK)

    Cabeceras: información del servidor, tipo de contenido, tamaño, etc.

    Cuerpo: los datos solicitados.

    Métodos de petición HTTP

    Los métodos indican qué acción se quiere realizar sobre el recurso

    Método: descripción

    GET: Solicita datos del servidor (no modifica nada).

    post: Envía datos al servidor (por ejemplo, un formulario)

    put: Actualiza un recurso existente en el servidor.

    Delete: Elimina un recurso del servidor.

    Head:Igual que GET, pero solo solicita las cabeceras.

    patch:Modifica parcialmente un recurso

    option:Solicita información sobre los métodos permitidos para un recurso.

    Códigos de respuesta HTTP

    1xx – Informativos

    100 Continue: El servidor recibió parte de la solicitud y el cliente debe continuar.

    2xx – Éxito 200 OK: Solicitud correcta. 201 Created: Recurso creado exitosamente. 204 No Content: Solicitud correcta, pero sin contenido en la respuesta.

    3xx – Redirección 301 Moved Permanently: El recurso se ha movido de forma permanente. 302 Found: Redirección temporal. 304 Not Modified: El recurso no ha cambiado desde la última solicitud.

    4xx – Errores del cliente 400 Bad Request: Solicitud mal formada. 401 Unauthorized: Requiere autenticación. 403 Forbidden: Acceso denegado. 404 Not Found: El recurso no existe. 405 Method Not Allowed: Método HTTP no permitido.

    5xx – Errores del servidor 500 Internal Server Error: Error general del servidor. 502 Bad Gateway: El servidor actuó como puerta de enlace y recibió una respuesta inválida. 503 Service Unavailable: El servidor no puede atender la solicitud (por mantenimiento o sobrecarga). 504 Gateway Timeout: El servidor no respondió a tiempo.

    En resumen

    HTTP es el protocolo base de la comunicación web. Funciona con un modelo cliente-servidor. Usa métodos de petición para indicar acciones. Devuelve códigos de respuesta que informan el resultado. Evoluciona constantemente para mejorar velocidad y seguridad.

    Puertos de red

    definicion

    Lospuertos TCP (Protocolo de Control de Transmisión) y UDP (Protocolo de Datagramas de Usuario)

    son puntos finales en una red que permiten a los dispositivos comunicarse entre sí. La principal diferencia radica en su enfoque: TCP es un protocolo orientado a la conexión.

    fiable y que garantiza la entrega de datos en orden, ideal para aplicaciones como el correo electrónico y la navegación web.

    UDP es un protocolo sin conexión, más rápido y eficiente, pero no garantiza la entrega ni el orden de los paquetes, lo que lo hace ideal para aplicaciones de streaming y juegos en línea

    TCP (Protocolo de Control de Transmisión)

    Orientado a la conexión: Establece una conexión entre el emisor y el receptor antes de enviar datos.

    Fiable: Asegura que todos los paquetes de datos lleguen correctamente y en el orden correcto, incluso retransmitiendo los paquetes perdidos.

    Lento: El establecimiento de la conexión y las confirmaciones de entrega hacen que el proceso sea más lento que con UDP.

    Uso común: Correo electrónico, navegación web (HTTP/HTTPS), transferencia de archivos.

    UDP (Protocolo de Datagramas de Usuario)

    Sin conexión: Envía paquetes de datos sin establecer una conexión previa.

    No fiable: No garantiza la entrega ni el orden de los paquetes; los paquetes perdidos no se retransmiten.

    Rápido: Al no tener la sobrecarga de la conexión y la confirmación, es mucho más rápido que TCP.

    Uso común: Streaming de video y audio, voz sobre IP (VoIP) y juegos en línea, donde la velocidad es más importante que la pérdida ocasional de datos.

    Puntos claves

    Ambos protocolos funcionan en la Capa de Transporte y utilizan números de puerto para dirigir los datos a la aplicación correcta en un dispositivo

    Un puerto puede ser utilizado por TCP o UDP, pero un servicio específico generalmente prefiere uno sobre el otro

    Por ejemplo, DNS utiliza ambos: UDP para la mayoría de las consultas y TCP para transferencias de zona más grandes.

    Los números de puerto son asignados a servicios específicos. El puerto 80 es para HTTP, el 443 para HTTPS y el 53 para DNS, entre otros.

    Tipos de puertos

    Protocolo HTTP Num de puerto 80 funcionamiento: Transferencia de páginas web no cifradas.

    Protocolo HTTPS Num de puerto 443 funcionamiento:Transferencia de páginas web seguras (HTTP sobre SSL/TLS).

    Protocolo SMTP Num de puertos 25, 465, 587 funcionamiento: Envío de correo electrónico (25 estándar, 465 con SSL, 587 con STARTTLS).

    Protocolo POP3 NUM de puerto 110, 995 funcionamiento: Recepción de correo electrónico (995 usa SSL/TLS).

    Protocolo IMAP NUm de puerto 143, 993 funcionamiento: Acceso a correo en servidor (993 usa SSL/TLS).

    Protocolo MIME No tiene puerto propio Funcionamiento: Es una extensión de formato del correo electrónico, no un protocolo de transporte. Funciona sobre SMTP/POP3/IMAP.

    Protocolo FTP Num de puertos 21 (control), 20 (datos) Funcionamiento: Transferencia de archivos; modo activo y pasivo.

    Protocolo FTPS Num de puertos 990 (implícito), 21 (explícito) Funcionamiento: FTP con seguridad SSL/TLS

    Protocolo SFTP Num de puerto 22 Funcionamiento: Transferencia de archivos segura sobre SSH.

    Protocolo BitTorrent Num de puerto 6881–6889 (recomendado) Funcionamiento: ntercambio P2P de archivos; los clientes pueden usar otros puertos configurables.

    Protocolo FastTrack Num de puerto 1214 Funcionamiento: Red P2P usada por Kazaa y Grokster.

    Protocolo IRC NUm de Puertos 6660–6669, 194, 7000 Funcionamiento: Chat en tiempo real (Internet Relay Chat).

    Protocolo XMPP NUm de puertos 5222 (no cifrado o STARTTLS), 5223 (SSL), 5269 (entre servidores) Funcionamiento: Mensajería instantánea y comunicación federada (Jabber).

    Protocolo MTProto Num de puertos 443, 80, 88, 5222 (varía según implementación) Funcionamiento: Protocolo propio de Telegram para mensajería segura; usa puertos comunes para evitar bloqueos.
