Tipologia de contenidos de internet

definicion

La tipologuia de contenidos de internetse refiere a la clasificacion de los diferentes tipos de informacion, formatos y materiales que se publican y circulan en la red. Esta clasificacion permite entender como se organiza la informacion y cuales son sus propocitos, audiencia y formatos.

Clasifique 4 tipos de tipoligias que considere importante

1.contenido informatico

Definicion:Tiene como objetivo informar o educar al usuario sebre temas especificos.

  • Noticias digitales
  • Artículos educativos
  • Tutoriales paso a paso
  • Entradas de blog

2. Contenido de Entretenimiento

Definicion:Busca divertir, emocionar y entretener a la persona o usuario.

  • Videos virales
  • Memes

  • video juegos

3. Contenido comercial o publicitario

Definición: Diseñado con fines de marketing, para promocionar productos o servicios.

  • Anuncios online
  • Contenido patrocinado
  • Descripciones de productos
  • Campañas de email marketing

4. Contenido Interactivo

Definicion: Permite la participacion del usuario mediante interaccion directa.

  • Quizzes y test online
  • Infografías interactivas
  • Juegos web

Definición general

definicion

El Portocolo HTTPes el protocolo que permite la comunicación entre un cliente (navegador) y un servidor web.

Su función principal es transferir información como páginas, imágenes o videos a través de Internet

Trabaja bajo el modelo cliente-servidor: el cliente hace una petición y el servidor envía una respuesta.

Estructura y funcionamiento

Estructura y funcionamiento

Definicion:Una comunicación HTTP se compone de dos partes.

Petición (Request) → enviada por el cliente. Contiene:

    Parte1

  • Metodo (Get,post,etc)
  • Url del recurso
  • Encabezados ( informacion adicional)
  • Cuerpo(opcional,cuando se envia datos)

Parte2

  • Codigo de estado (ej.200 ok,404 Not Fonund)
  • Encabezasos del servidor
  • Cuerpo con el contenido solicitado

    • Estructura de una petición HTTP

    Línea de petición: indica el método (GET), el recurso (/index.html) y la versión del protocolo (HTTP/1.1)

    Cabeceras (headers): proporcionan información adicional (navegador, tipo de contenido, idioma, etc.).

    Cuerpo (body): opcional; se usa para enviar datos (por ejemplo, en formularios o JSON)

    ejemplo

    Línea de estado: contiene la versión de HTTP, el código de estado y su descripción (por ejemplo, 200 OK)

    Cabeceras: información del servidor, tipo de contenido, tamaño, etc.

    Cuerpo: los datos solicitados.

    Métodos de petición HTTP

    Los métodos indican qué acción se quiere realizar sobre el recurso

    Método: descripción

    GET: Solicita datos del servidor (no modifica nada).

    post: Envía datos al servidor (por ejemplo, un formulario)

    put: Actualiza un recurso existente en el servidor.

    Delete: Elimina un recurso del servidor.

    Head:Igual que GET, pero solo solicita las cabeceras.

    patch:Modifica parcialmente un recurso

    option:Solicita información sobre los métodos permitidos para un recurso.

    Códigos de respuesta HTTP

    1xx – Informativos

    100 Continue: El servidor recibió parte de la solicitud y el cliente debe continuar.

    2xx – Éxito 200 OK: Solicitud correcta. 201 Created: Recurso creado exitosamente. 204 No Content: Solicitud correcta, pero sin contenido en la respuesta.

    3xx – Redirección 301 Moved Permanently: El recurso se ha movido de forma permanente. 302 Found: Redirección temporal. 304 Not Modified: El recurso no ha cambiado desde la última solicitud.

    4xx – Errores del cliente 400 Bad Request: Solicitud mal formada. 401 Unauthorized: Requiere autenticación. 403 Forbidden: Acceso denegado. 404 Not Found: El recurso no existe. 405 Method Not Allowed: Método HTTP no permitido.

    5xx – Errores del servidor 500 Internal Server Error: Error general del servidor. 502 Bad Gateway: El servidor actuó como puerta de enlace y recibió una respuesta inválida. 503 Service Unavailable: El servidor no puede atender la solicitud (por mantenimiento o sobrecarga). 504 Gateway Timeout: El servidor no respondió a tiempo.

    En resumen

    HTTP es el protocolo base de la comunicación web. Funciona con un modelo cliente-servidor. Usa métodos de petición para indicar acciones. Devuelve códigos de respuesta que informan el resultado. Evoluciona constantemente para mejorar velocidad y seguridad.
